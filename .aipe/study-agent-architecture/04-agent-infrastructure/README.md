# 04 — Agent infrastructure

**Anchor:** single-agent + multi-agent (both).

The cross-cutting disciplines that separate a demo from a shipped system.
flattr has no agent, so most of this is study material — but two files have
real flattr anchors:

- `03-tool-calling-and-mcp.md` — ★ flattr's four router functions are
  pre-cut tools (the seam).
- `04-agent-evaluation.md` — ★ `bench/` is the trajectory-eval attachment
  point (it's already a stage-comparison harness).

`05-guardrails-and-control.md` carries the budget exit from the agent-loop
skeleton as a control envelope.
